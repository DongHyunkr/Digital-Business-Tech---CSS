Data Fabric

정의

: 공유 데이터로 구성된 종단 간 데이터 통합 및 관리 솔루션.

∙ 직물처럼 교차되어 엮어진 연결망, 데이터 패브릭

장∙단점

  • 장점
    • - 데이터 검색 및 수집에서 전달 및 소비에 소요되는 시간을 단축
    - 자동화로 인해 시간을 절약하고 오류 및 규정 준수 문제의 위험을 감소시킬 수 있다.
    - 민감한 데이터를 보호하며 엔터프라이즈 데이터 관리
  • 단점
    • - 유지보수를 위해 가상화, 카탈로그 등 기술을 잘 아는 전문 인력을 고용해야 한다.
    - 여러 기술에 걸친 단일 통합 플랫폼을 제공하므로 서로 다른 데이터와 시스템을 통합하는 중앙 집권식 프로세스는 때때로 데이터를 다르게 포맷해 버린다.

기술 사용 사례
  • 정책 기반 데이터 관리
    • - 데이터 추상화를 통해 데이터 저장소 비용과 성능 간의 적절한 균형.
    - 고성능 워크로드는 더욱 빠르고 응답성이 뛰어난 저장소가 합리적.
  • 자동화된 메타데이터 개선
    • - 데이터는 종종 더욱 완전하고 사용 가능한 메타데이터 집합을 생성할 수 있는 단서
    - 지리적 위치 좌표가 있다면 우편번호, 도시, 국가를 추가할 수 있고, 싱품 코드를 이용할 수 있다면 상품 정보를 입력 가능.
  • 데이터 전환 영역
    • - 표준화할 수 있는 많은 데이터 파이프라인 공통으로 전환
    - 수십 개의 데이터 파이프라인을 각각 중복 제거와 병합 루틴을 시행하지 않고도 모든 데이터가 중복 제거될 수 있도록 중복 제거 영역에 데이터를 배치.
  • 금융 서비스
    • - 데이터 패브릭은 데이터 관리 아키텍처로, 중앙 집중식 엑세스와 회사 전체의 당일 통합 데이터 보기를 제공.
    - 내장된 거버넌스는 금융 서비스와 같이 규제가 엄격한 산업에 중요한 데이터 보안에 도움.
